# CountdownApp 领域上下文

本文记录代码中需要保持一致的核心术语，避免界面、会话和提醒模块对同一概念产生不同解释。

## 核心术语

- **专注周期（Focus session）**：从用户点击开始到专注结束、停止或退出的完整会话；暂停时间不计入有效专注时间。
- **随机提醒（Reminder）**：由调度器在安全随机区间内选出的微休息事件。一次提醒必须最终得到跳过、自动结束或注意力反馈之一。
- **微休息（Microbreak）**：短横幅或遮罩倒计时。开始铃声在出现时播放，只有倒计时自然结束才播放回归专注铃声。
- **注意力反馈（Attention feedback）**：仍在任务、刚才走神、正在心流三种反馈；反馈会调整下一次提醒间隔，并结束当前提醒。
- **大休息（Long break）**：专注周期自然结束后的独立休息阶段，不产生随机提醒。
- **V2 阶段（V2 phase）**：注意力锚定期、深度专注期、疲劳维护期；每个阶段拥有独立基础随机区间。
- **提醒视图（Reminder view）**：管理提醒窗口、多屏覆盖、焦点、倒计时刷新和关闭结果的界面模块；不修改专注会话，也不决定铃声策略。
- **运行视图（Runtime view）**：管理专注或大休息期间的仪表盘、背景音卡片、操作按钮与响应式窗口布局；只接收结构化显示数据和用户动作回调。
- **设置表单（Settings form）**：保存设置页变量，负责配置载入、选项映射、用户输入解析和完整设置构造；错误信息应指出具体字段。
- **V2 节律编辑器（V2 settings dialog）**：编辑三个 V2 阶段边界与随机区间的单实例窗口；直接绑定设置表单，但不构造会话设置。
- **设置视图（Settings view）**：管理主设置页布局、滚动、更多设置、算法摘要、快捷键控件与窗口尺寸；业务动作通过回调交给应用层。

## 关键边界

- `FocusSession` 是会话时间与状态的唯一事实来源。
- `ReminderView` 只返回结构化结果；`CountdownApp` 把结果映射为会话反馈和音频动作。
- `RuntimeView` 不读取 `FocusSession`；`CountdownApp` 将会话状态转换为 `RuntimeDisplay` 后再渲染。
- `SettingsForm` 是设置页输入的唯一解析与校验入口；界面层不得重复解释分钟、选项标签或自定义音频路径。
- `V2SettingsDialog` 只管理窗口与字段布局，关闭后通知设置页刷新摘要。
- `SettingsView` 可以读写 `SettingsForm` 的显示变量，但不得保存配置、启动会话或控制音频设备。
- GUI 显示不得通过“每秒减一”计算业务时间，应读取基于 `time.monotonic()` 的剩余时间。
- 随机提醒使用操作系统密码学随机源；它不是物理硬件真随机数。
