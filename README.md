# 随机专注计时器 3.0

随机专注计时器是一个面向长时间专注的随机打断式计时器。它支持传统的统一随机间隔，也支持“前密、中疏、后密”的 V2 三阶段算法。

## 主要功能

- 30、60、90 分钟快捷时长和自定义时长；
- 使用操作系统级安全随机源生成不可预测的提醒间隔；
- Classic 自定义统一随机间隔；
- V2 包含注意力锚定、深度专注和疲劳维护阶段；
- 可选自适应反馈：提醒时选择“仍在任务”“刚才走神”或“正在心流，延后下次提醒”，自动微调下一次间隔；
- 平衡和强干预两种提醒预设；Classic 与 V2 疲劳期分别提供独立的全屏开关，均默认关闭；
- 全屏提醒覆盖完整的多显示器虚拟桌面并保留 Esc 退出；平衡预设使用半透明全屏，强干预使用全黑全屏并主动获取焦点；
- 暂停、停止、系统托盘和电脑睡眠恢复；
- 专注与大休息共用一个单调时钟状态模型，暂停和恢复不会依赖 GUI 回调次数；
- 默认遮蔽精确的下一次提醒时刻，也可在“更多设置”中主动显示剩余时间；
- 可选置顶悬浮计时条，复用主计时刷新，支持拖动和位置记忆，且不会占用任务栏或 Alt+Tab；显示器布局变化时会自动恢复到最近的可见工作区；右键或 × 隐藏后，可在专注运行或暂停期间通过托盘“显示悬浮计时”恢复；
- 可选 Windows 原生全局快捷键，默认使用 `Alt+Shift+P` 暂停/继续、`Alt+Shift+O` 显示/隐藏主界面，也可自定义；
- 可选择关闭主窗口时隐藏到托盘，或直接退出；
- 开机启动可选择显示主界面或静默到托盘，也可完全关闭；
- 单实例运行；重复启动不会创建第二套计时器，指向旧安装目录的开机启动项会自动清理；
- EXE、主窗口、子窗口和系统托盘使用统一的多尺寸应用图标，在浅色与深色任务栏下保持可辨识；
- 独立的“微休息开始铃”和“回归专注铃”，默认分别为 `0.wav` 和 `1.wav`，均可从五个内置提示音中选择或使用自定义 WAV、OGG、MP3；
- 专注期间可组合三层背景音：白／粉红／棕／灰基础噪音、内置的风雨雷暴／雨声 OGG 录音，以及 174–963 Hz 的九种可选 Solfeggio 频率；
- 运行页提供可折叠的背景音控制条，可在不重置计时的情况下即时更换基础噪音、环境录音、Solfeggio 和总音量；修改会保存为下一轮默认值，暂停期间调整不会恢复播放；
- 新背景音组合在工作线程生成，GUI 不会因 PCM 合成而卡顿；快速连续切换时只播放最后一次选择；
- 可关闭休息倒计时；自动结束时播放回归铃，手动跳过时不播放；
- 高频设置采用单列布局，提醒强度、声音和系统行为收纳在“更多设置”，V2 细节使用独立窗口；
- 设置页支持滚动条和鼠标滚轮，适配高 DPI 与较小屏幕；
- 提醒小窗按中文内容和当前屏幕自动确定尺寸；空间不足时反馈按钮自动换行，不依赖固定窗口宽高；
- 专注运行页使用独立紧凑窗口，背景音控件展开和收起时自动调整高度；返回设置页后恢复设置窗口尺寸；
- 计时显示仅在文字变化时重绘，减少 Tkinter 无效刷新和后台 CPU 唤醒；
- 音频失败重试和 Windows 系统提示音降级；
- 单一音频引擎统一管理提示铃和背景音；提示铃播放时自动降低背景音，结束后恢复；
- 专注结束后由用户确认是否开始大休息；
- 便携式配置：`settings.json` 和 `Logs\countdown.log` 都写在程序安装目录。

配置缺少新版字段时会用安全默认值补全，schema v1 会原位迁移并保留用户偏好；配置内容损坏时，原文件会改名为
`settings.invalid.json`（已有备份时自动编号），随后使用默认设置启动。

“系统安全随机”使用 Python `secrets.SystemRandom`，不代表外接硬件产生的物理真随机。

基础噪音和 Solfeggio 在本地生成短 PCM 缓冲；风雨雷暴与雨声使用项目随附的 OGG 录音。多层通过独立声道循环并按等功率分配总音量。Solfeggio 频率只是声音选项，本项目不对其作医疗、治疗或提升认知效果的承诺。

全局快捷键默认关闭，使用 Windows `RegisterHotKey` 并复用现有 GUI 命令轮询，不增加第三方依赖或后台监听线程。支持 `Ctrl`、`Alt`、`Shift`、`Win` 修饰键与 `A–Z`、`0–9`、`F1–F12`、`Space` 组合；必须包含至少一个修饰键。注册冲突、重复动作或格式错误时会自动关闭并提示。启用后输入框锁定，需要先关闭快捷键再修改。

## 算法

### Classic

每次在用户设置的最小和最大间隔之间重新抽取。小于 60 分钟的任务默认使用 Classic。

### V2

默认按总时长分为：

| 阶段 | 90 分钟示例 | 默认间隔 |
| --- | ---: | ---: |
| 注意力锚定期 | 0–30 分钟 | 4–7 分钟 |
| 深度专注期 | 30–65 分钟 | 10–15 分钟 |
| 疲劳维护期 | 65–90 分钟 | 5–8 分钟 |

不足 60 分钟时仍可手动启用 V2；不足 15 分钟时不提供 V2。

### 什么时候会全屏

全屏提醒默认关闭。启用“休息倒计时”时，Classic 提醒和 V2 的疲劳维护期默认使用小提醒窗；“更多设置 → 提醒与铃声”分别提供“Classic 全屏提醒”和“V2 疲劳期全屏提醒”，两个开关互不影响。V2 注意力锚定期始终使用小窗，深度专注期始终优先使用系统通知。全屏模式下，平衡预设为半透明遮罩，强干预预设为全黑遮罩并获取焦点。关闭“休息倒计时”后仅使用声音、通知或短暂提示，不显示倒计时窗口。

### 自适应反馈（可选）

自适应反馈默认关闭，Classic 与 V2 都可使用。每次仍先从当前阶段的基础区间进行系统安全随机抽样，再按反馈调整下一次间隔：

| 条件 | 下一次间隔倍率 |
| --- | ---: |
| 连续 2 次“刚才走神” | 75% |
| 连续 3 次“仍在任务” | 125% |
| “正在心流，延后下次提醒” | 150% |
| 跳过、超时或尚未达到连续阈值 | 100% |

阶段边界和专注结束时间始终优先。反馈计数只保存在当前专注周期内，用于运行页摘要和下一次调度；不会写入数据库、上传或建立长期行为档案。

完整设计和验收标准见 [V2 重构设计](docs/V2_REDESIGN.md)。

## 开发运行

建议使用 Python 3.14 x64：

```powershell
python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements-dev.txt
.\.venv\Scripts\python countdown_app.py
```

也可以使用模块入口：

```powershell
.\.venv\Scripts\python -m countdownapp
```

## 测试

```powershell
.\.venv\Scripts\python -m unittest discover -v
```

图标需要调整时，可运行 `.\.venv\Scripts\python scripts\generate_icon.py`，统一重新生成透明 PNG 和包含 16–256 像素帧的 Windows ICO。

核心计时、阶段、随机源和配置层不依赖 Tkinter 窗口，测试通过假时钟和假随机源验证行为。

## Windows 打包

当前 spec 生成免安装的单文件 `onefile` 程序：

```powershell
.\build.cmd
```

输出为 `dist\CountdownApp.exe`，五个提示音、两个环境录音、托盘图标、Python 与第三方依赖均封装在 EXE 中。
首次运行会在 EXE 同目录自动创建 `settings.json` 和 `Logs`；`.pyinstaller-build` 是不完整的打包中间目录，不能从中启动程序。
重新构建会保留 `dist` 中已有的 `settings.json` 和 `Logs`。
单文件启动时会先解压运行组件到系统临时目录，因此首次启动通常比目录版稍慢。
构建会排除托盘图标不需要的 AVIF、WebP、色彩管理等 Pillow 可选解码器，以减少单文件体积和解压开销；包含两段环境录音和多尺寸图标后的当前构建约 26.8 MB。
CI 会在 Windows Python 3.14 上构建单文件，把它复制到空目录启动，并验证版本、配置和日志自动生成。
生成的个人测试构建未做数字签名，Windows 可能显示未知发布者；公开分发前应使用可信代码签名证书签名。
